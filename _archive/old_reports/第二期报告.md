# Director Tool Calling 调研报告

> 调研日期: 2026-03-01
> 目标: 评估当前 Director 能力，规划 LLM-driven skill selection 实现

---

## 1. 现状总结

### 1.1 当前架构

```
用户消息
    │
    ▼
DirectorAgent
    │
    ├─ classify_intent()      ← 规则匹配（关键词 → TaskIntent 枚举）
    │
    ├─ detect_workflow()      ← 规则匹配（intent → WorkflowDefinition）
    │
    ├─ [有 workflow]
    │   └─ _execute_phase()
    │       └─ _execute_tool()  ← 硬编码 if/else 分发
    │
    └─ [无 workflow]
        └─ _process_without_workflow()
            └─ LLM 生成文本（不选择工具）
```

### 1.2 核心问题

| 问题 | 描述 |
|------|------|
| **工具选择是硬编码** | `_execute_tool` 使用 if/else 链，不是动态选择 |
| **LLM 只生成文本** | LLM 用于生成对话，不参与工具选择决策 |
| **Skill 系统未集成** | `SkillRegistry` 和 `ToolExecutor` 存在但 Director 不使用 |
| **无文件读取能力** | Director 只能通过 `_load_context` 加载特定数据类型 |

## 3. 缺失能力

### 3.1 LLM 驱动的工具选择

| 缺失 | 描述 |
|------|------|
| **Tool Schema** | 没有将 Skills 转换为 LLM 可理解的工具定义 |
| **Tool Calling API** | LLMClient 不支持 function calling |
| **JSON Mode** | 没有强制 JSON 输出，依赖脆弱的正则解析 |
| **工具选择 Prompt** | 没有 prompt 让 LLM 从可用工具中选择 |

### 3.2 Director 与 Skill 系统集成

| 缺失 | 描述 |
|------|------|
| **SkillRegistry 集成** | Director 不知道 SkillRegistry 的存在 |
| **ToolExecutor 集成** | Director 无法使用通用文件操作工具 |
| **动态 Skill 发现** | 无法在运行时发现新注册的 Skills |

### 3.3 上下文收集

| 缺失 | 描述 |
|------|------|
| **LLM 决定上下文** | 上下文收集是硬编码的，LLM 不能决定需要什么 |
| **文件读取** | Director 不能主动读取任意文件 |
| **智能上下文裁剪** | 没有基于相关性的上下文选择 |

## 4. 实现计划

### 4.1 架构目标

```
用户消息
    │
    ▼
DirectorAgent
    │
    ├─ SkillRegistry.get_skills_prompt()  ← 获取可用工具描述
    │
    ├─ LLM.decide_tools(user_message, available_tools)
    │       │
    │       ▼
    │   {
    │     "reasoning": "用户想写第三章，需要...",
    │     "selected_tools": [
    │       {"name": "query_outline", "args": {"chapter_id": "ch_003"}},
    │       {"name": "write_chapter", "args": {"chapter_id": "ch_003"}}
    │     ]
    │   }
    │
    ├─ ToolExecutor.execute(tool_name, args)  ← 执行选中的工具
    │
    └─ LLM.generate_response(tool_results)    ← 生成最终回复
```

### 4.2 具体实现步骤

#### Phase 1: LLM Client 增强

1. **添加 JSON Mode 支持**
   ```python
   # tools/llm/client.py
   def complete(self, messages, model, max_tokens, response_format=None):
       kwargs = {"messages": messages, "model": model, "max_tokens": max_tokens}
       if response_format:
           kwargs["response_format"] = response_format
       return litellm.completion(**kwargs)
   ```

2. **添加 Tool Calling 支持**（可选，取决于模型）
   ```python
   def complete_with_tools(self, messages, tools, tool_choice="auto"):
       # 使用 LiteLLM 的 function calling 支持
       return litellm.completion(
           messages=messages,
           tools=tools,
           tool_choice=tool_choice,
       )
   ```

               "parameters": {

                   }
               }
           }
       }

   def executor_tool_to_schema(tool_name: str, tool_func: Callable) -> dict:
       # 从 docstring 和签名生成 schema
   ```

2. **聚合所有可用工具**
   ```python
   def get_all_tool_schemas() -> List[dict]:
       schemas = []
       # 从 SkillRegistry
       for skill in skill_registry.list_all():
           schemas.append(skill_to_tool_schema(skill))

#### Phase 3: Director Prompt 增强

1. **新增 `director_tool_selection` prompt**
   ```python
   @staticmethod
   def director_tool_selection(
       user_message: str,
       available_tools: List[dict],
       context_summary: str,
   ) -> List[dict]:
       return [
           {"role": "system", "content": f"""
   你是 OpenWrite 的主控 Agent。根据用户请求，选择合适的工具执行。

   可用工具:
   {json.dumps(available_tools, ensure_ascii=False, indent=2)}

   当前上下文:
   {context_summary}

   输出 JSON 格式:
   ```json
   {{
     "reasoning": "分析用户意图...",
     "selected_tools": [
       {{"name": "tool_name", "args": {{"arg1": "value1"}}}}
     ],
     "response_to_user": "可选：直接回复用户的话"
   }}
   ```
   """},
           {"role": "user", "content": user_message},
       ]
   ```

#### Phase 4: Director 集成

1. **在 DirectorAgent 中集成 SkillRegistry 和 ToolExecutor**
   ```python
   class DirectorAgent:
       def __init__(self, ...):
           # 现有初始化...
           self._skill_registry = get_skill_registry()
           self._tool_executor = None  # 延迟初始化（需要 novel_id）

       def _get_tool_executor(self, novel_id: str) -> ToolExecutor:
           return ToolExecutor(
               project_root=Path.cwd(),
               novel_id=novel_id,
           )
   ```

2. **新增 LLM 驱动的工具选择方法**
   ```python
   def _select_tools_with_llm(
       self,
       user_message: str,
       context: Dict[str, Any],
   ) -> Dict[str, Any]:
       # 1. 获取可用工具
       tools = get_all_tool_schemas()

       # 2. 调用 LLM 选择工具
       messages = PromptBuilder.director_tool_selection(
           user_message=user_message,
           available_tools=tools,
           context_summary=self._build_context_summary(context),
       )

       response = self._llm_client.complete(
           messages=messages,
           response_format={"type": "json_object"},
       )

       # 3. 解析结果
       return json.loads(response.content)
   ```

3. **实现工具执行循环**
   ```python
   def _execute_tool_loop(
       self,
       tool_selections: List[dict],
       context: Dict[str, Any],
   ) -> List[dict]:
       results = []
       executor = self._get_tool_executor(context.get("novel_id", ""))

       for tool in tool_selections:
           result = executor.execute(tool["name"], tool["args"])
           results.append({
               "tool": tool["name"],
               "result": result,
           })

       return results
   ```

#### Phase 5: 整合到主流程

修改 `process_request_with_workflow` 和 `_process_without_workflow`:
- 当 LLM 可用时，使用 `_select_tools_with_llm` 选择工具
- 当 LLM 不可用时，fallback 到现有规则引擎
