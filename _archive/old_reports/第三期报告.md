## 一、架构重大变化

### 1.1 新的 Skill 模块系统 ✅

第一期报告中用户提出的期望：
> 主AI决策能知道用户要做什么，比如用户说要由大纲写章节，Director就去记录如何上下文组装的文件看怎么进行，用哪些tool

**已实现**：完整的 Skill 模块架构

```
skills/
├── skill.py              # Skill 数据模型
├── skill_registry.py     # 功能注册表（触发器匹配、优先级管理）
├── skill_loader.py       # 文件系统加载器
├── tools/
│   └── executor.py       # 统一工具执行器（13个工具）
├── outline/              # 大纲功能模块
│   ├── SKILL.md          # 功能指令文档
│   ├── tools/            # parser, serializer, validator
│   └── prompts/          # 生成提示词模板
├── writing/              # 写作功能模块
│   ├── SKILL.md
│   ├── tools/            # beat_generator, draft_generator
│   ├── workflows/        # chapter_writing.yaml
│   └── templates/        # 节拍/段落模板
├── style/                # 风格功能模块
├── character/            # 角色功能模块
├── foreshadowing/        # 伏笔功能模块
├── world/                # 世界观功能模块
└── project/              # 项目管理功能模块
```

**核心能力**：
- `SkillRegistry.match_trigger(text)` — 动态意图匹配，替代硬编码关键词
- `ToolExecutor.execute(tool_name, args)` — 统一工具调用接口
- 优先级覆盖机制：project > user > builtin

### 1.2 新 Agent 实现 ✅

| 新 Agent | 文件 | 行数 | 职责 |
|----------|------|------|------|
| **SkillBasedDirector** | `director_v2.py` | 1142 | 基于 Skill 的主控 Agent |
| **PipelineSimulatorV2** | `pipeline_v2.py` | 507 | 人工审核介入的新流水线 |

**SkillBasedDirector 核心改进**：
- 外部提示词加载：`config/main_prompt.md`
- SkillRegistry 集成：`skill_registry.match_trigger()`
- ToolExecutor 集成：`tool_executor.execute()`
- 工作流检测与执行
- 向后兼容：`DirectorAgent = SkillBasedDirector`

**PipelineSimulatorV2 流程**：
```
Director (上下文组装)
    ↓
Writer (章节生成)
    ↓
Reviewer (逻辑检查，只读)
    ↓
User Review (人工审核确认) ← 新增！
    ↓
Stylist (风格润色，可选)
```

**关键特性**：
- Mild issues 触发用户确认，不自动通过
- Severe errors 触发自动重写（最多 3 次）
- 阶段化执行支持 Web UI

### 1.3 新模型定义 ✅

| 模型 | 文件 | 用途 |
|------|------|------|
| `WorkflowDefinition` | `workflow.py` | 工作流定义（阶段、触发条件、前置条件） |
| `WorkflowPhase` | `workflow.py` | 工作流阶段（工具配置、上下文需求） |
| `WorkflowState` | `workflow.py` | 运行时状态追踪 |
| `IntentDecision` | `intent.py` | 意图识别结果（置信度、实体引用） |
| `DirectorResponse` | `intent.py` | 统一响应格式（工作流状态、建议操作） |
| `ConversationSession` | `intent.py` | 对话会话（消息历史、工作流上下文） |

### 1.4 Tool Schema 生成系统 ✅

**新文件**：`tools/llm/tool_schema.py`（492 行）

将 Skills 和 ToolExecutor 工具转换为 OpenAI-compatible schemas：

```python
# 将 Skill 转为 function schema
skill_to_tool_schema(skill: Skill) -> Dict[str, Any]

# 将 ToolExecutor 工具转为 function schema  
executor_tool_to_schema(tool_name: str, tool_func: Callable) -> Dict[str, Any]

# 聚合所有可用工具 schemas
get_all_tool_schemas(skill_registry, tool_executor) -> List[Dict]

# 格式化为 prompt 文本
format_tools_for_prompt(schemas, format_type="markdown") -> str
```

## 六、新架构总览

```
用户消息
    │
    ▼
SkillBasedDirector
    │
    ├─ SkillRegistry.match_trigger()  ← 动态意图匹配
    │
    ├─ WorkflowRegistry.match_workflow()  ← 工作流检测
    │
    ├─ [有工作流]
    │   └─ _execute_workflow()
    │       └─ ToolExecutor.execute()  ← 统一工具调用
    │
    └─ [无工作流]
        └─ _process_without_workflow()
            └─ LLM 生成响应 / 通用对话
```

**三层工具体系**：

1. **Skills（功能模块）**：outline, writing, style, character, world, foreshadowing, project
2. **Tools（底层工具）**：read_file, write_file, query_*, search_content 等 13 个
3. **Workflows（工作流）**：outline_creation, chapter_writing, style_selection 等 5 个
