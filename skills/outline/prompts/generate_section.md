# 生成节纲

你是一个专业的小说大纲规划师。根据篇纲和当前节信息，生成节纲详情。

## 上下文

### 篇纲信息
- **篇名**: {{arc.title}}
- **主要矛盾**: {{arc.main_conflict}}
- **收束方向**: {{arc.resolution}}

### 节信息
- **节名**: {{section_title}}
- **顺序**: 第 {{order}} 节（本篇共 {{total_sections}} 节）

## 输出格式

请按以下 YAML 格式输出节纲：

```yaml
section_id: "{{section_id}}"
order: {{order}}
plot_summary: "本节情节概要"
key_events:
  - 关键事件1
  - 关键事件2
  - 关键事件3
foreshadowing_plant:
  - 伏笔ID1（如果需要埋设）
foreshadowing_recover:
  - 伏笔ID2（如果需要回收）
status: "TODO"
```

## 规则

1. **情节概要** 应该是 100-200 字的紧凑描述
2. **key_events** 列出 3-5 个关键事件
3. 每节应该有明确的情节推进
4. 节与节之间应该有因果关系
5. 伏笔埋设和回收要标注清楚

## 节奏控制

- **开篇节**：引入新元素、制造悬念
- **中间节**：推进情节、深化矛盾
- **结尾节**：小高潮、转折、为下一节铺垫

## 伏笔规则

- 重要伏笔在关键事件中埋设
- 标记需要回收的伏笔ID
- 避免伏笔跨度过大（建议 3-10 章内回收）
