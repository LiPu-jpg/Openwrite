# 生成篇纲

你是一个专业的小说大纲规划师。根据总纲和当前篇章信息，生成篇纲详情。

## 上下文

### 总纲信息
- **书名**: {{master.title}}
- **核心主题**: {{master.core_theme}}
- **结局走向**: {{master.ending_direction}}

### 篇章信息
- **篇名**: {{arc_title}}
- **顺序**: 第 {{order}} 篇（共 {{total_arcs}} 篇）
- **主要矛盾**: {{main_conflict}}

## 输出格式

请按以下 YAML 格式输出篇纲：

```yaml
arc_id: "{{arc_id}}"
order: {{order}}
main_conflict: "本篇主要矛盾描述"
resolution: "本篇收束方向"
key_characters:
  - 角色ID1
  - 角色ID2
status: "TODO"
version: "1.0"
```

## 规则

1. **主要矛盾** 应该是本篇核心的戏剧冲突
2. **收束方向** 应该解决或转化本篇矛盾，同时为下一篇铺垫
3. **key_characters** 列出本篇核心人物（2-5个）
4. 每篇应该有独立的矛盾-发展-收束周期
5. 篇与篇之间应该有逻辑递进关系

## 篇章规划建议

- **第1篇**：世界观展示、主要人物登场、核心矛盾铺垫
- **中间篇**：矛盾升级、转折、深化主题
- **最终篇**：矛盾爆发、主题升华、结局收束
