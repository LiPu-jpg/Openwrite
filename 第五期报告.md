## 一、问题诊断与修复

### 1.1 问题现象

用户反馈 `/chat` 页面会话历史存在以下问题：
- 会话列表可见但点击无响应
- 删除按钮点击无效果
- 所有会话显示 0 条消息

### 1.2 根因分析

经过系统性排查，发现 **5 个关联 bug**：

| # | 文件 | 问题 | 影响 |
|---|------|------|------|
| 1 | `chat.html:353` | `api.delete()` 不存在，应为 `api.del()` | 删除按钮失效 |
| 2 | `chat.html:387-396` | `addMessageToUI()` 构建了 HTML 但从未赋值给 `div.innerHTML` | 点击会话看不到历史 |
| 3 | `director_v2.py:982-989` | `process_request()` 只保存用户消息，从未保存 agent 回复 | message_history 为空 |
| 4 | `chat.html:351,370` | `deleteSession` 和 `switchSession` 未暴露到 `window` 对象 | HTML 内联事件无法调用 |
| 5 | `director_v2.py:987` | 空会话也会保存到磁盘，产生大量 0 条记录 | 污染会话列表 |

### 1.3 修复方案

**chat.html 修复**：
```javascript
// 修复1: api.delete → api.del
api.del('/api/sessions/' + sessionIdToDelete)

// 修复2: addMessageToUI 添加 innerHTML 赋值
div.innerHTML = html;
chatMessages.appendChild(div);

// 修复3: 暴露函数到 window 对象
window.deleteSession = function(sessionIdToDelete) { ... }
window.switchSession = function(newSessionId) { ... }
```

**director_v2.py 修复**：
```python
# 修复4: 在 process_request 返回前统一保存 agent 回复
if response.message:
    session.add_message("assistant", response.message)
    # 只有有实际对话内容才保存到磁盘
    if len(session.message_history) >= 2:
        self._save_session(session.session_id)
```

**main.css 修复**：
```css
/* 修复5: 会话列表支持滚动 */
.ide-explorer #sec-sessions {
    flex-shrink: 1;
    min-height: 0;
    overflow: hidden;
}
.ide-explorer #sec-sessions .explorer-list {
    overflow-y: auto;
    flex: 1;
}
```

## 四、会话管理系统

### 4.1 会话生命周期

```
用户发送消息
    │
    ▼
process_request() 
    │
    ├─ 获取/创建 session
    ├─ 添加用户消息
    ├─ 意图识别 + 工作流执行
    ├─ 添加 agent 回复
    │
    └─ 检查是否需要保存
        │
        ├─ message_history >= 2 → 保存到磁盘
        └─ message_history < 2 → 仅内存，不持久化
```

### 4.2 会话存储结构

```
data/sessions/
├── {session_id_1}.json    # 有实际对话的会话
├── {session_id_2}.json
└── ...
```

每个会话文件包含：
- `session_id`: 会话标识
- `novel_id`: 关联的小说项目
- `message_history`: 消息历史（用户 + agent）
- `created_at` / `updated_at`: 时间戳

### 4.3 前端交互

```javascript
// 会话列表加载
loadSessions() → GET /api/sessions

// 会话切换
switchSession(id) → GET /api/sessions/{id} → 渲染 message_history

// 会话删除
deleteSession(id) → DELETE /api/sessions/{id}
```
