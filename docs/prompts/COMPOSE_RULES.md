# 风格合成规则 (Style Composition Rules)

> 本文档定义如何将三层风格文档合成为最终生成指令。

## 三层架构

```
craft/          → 通用写作技法（跨作品通用）
styles/{id}/    → 作品风格指纹（每部作品独立）
novels/{id}/    → 作品设定（仅限本作品，由主系统注入）
```

## 合成优先级

```
用户偏好覆盖 > 作品设定（硬性约束） > 作品风格（核心约束） > 通用技法（可选参考）
```

- **作品设定**：角色一致性、术语、世界观规则、禁忌清单 → 不可违反
- **作品风格**：叙述声音、幽默体系、节奏偏好、语言风格 → 应当遵循，用户可调整
- **通用技法**：场景结构、信息揭示策略、张力模型 → 作为工具箱，按需选用

## 合成流程

### 1. 加载阶段

```
输入：
  - 用户选择的作品风格（可选多个，可不选）
  - 当前作品的设定文件（由 OpenWrite 主系统提供）
  - 用户的偏好覆盖（可选）

加载顺序：
  1. craft/ 全部加载为"可选工具箱"
  2. styles/{selected}/ 加载为"风格约束"
  3. novels/{current}/ 加载为"硬性约束"（由主系统提供）
  4. 用户偏好覆盖应用
```

### 2. 冲突解决

| 冲突类型 | 解决方式 |
|----------|----------|
| 作品设定 vs 作品风格 | 作品设定优先（如：作品禁止粗口，即使风格允许） |
| 多作品风格混合 | 用户指定权重，或取交集 |
| 通用技法 vs 作品风格 | 作品风格优先（如：风格偏好短章节，即使通用技法建议长章节） |
| 用户偏好 vs 任何层 | 用户偏好始终优先 |

### 3. 输出格式

合成后输出到 `composed/{novel_id}_final.md`，结构：

```markdown
# 最终风格文档：{作品名}

## 来源
- 作品风格：{style_id}（权重：100%）
- 通用技法：已加载
- 作品设定：{novel_id}

## 硬性约束（不可违反）
{从 novels/ 层提取的禁忌清单和一致性规则}

## 风格约束（应当遵循）
{从 styles/ 层提取的核心风格要素}

## 可选技法（按需使用）
{从 craft/ 层提取的相关场景模板和策略}

## 用户覆盖
{用户的自定义调整}
```

## 多作品工作流

### 风格复用

```
跑完《术师手册》 → styles/术师手册/
跑完《我师兄》   → styles/我师兄/

用户可以：
1. 选择单一作品风格
2. 混合多个作品风格（指定权重）
3. 只用通用技法（不加载任何作品风格）
4. 从多个作品的 fingerprint.md 中浏览"风格推荐"
```

### 目录结构

```
styles/
├── 术师手册/
│   ├── fingerprint.md
│   ├── voice.md
│   └── ...
├── 我师兄/
│   └── ...
└── ...
```

## Reader/Director 循环适配

### Reader 阶段

Reader 输出时需要标注每个发现属于哪一层：

```markdown
## 发现：{模式名称}
- **层级**：craft / style / novel
- **判断依据**：{为什么归入这一层}
- **内容**：{具体描述}
```

### Director 阶段

Director 差异分析时需要区分：

```markdown
## 偏差分析
### 风格偏差（style 层）
- {偏差描述} → 更新 styles/{id}/{file}.md

### 设定偏差（novel 层）
- {偏差描述} → 更新 novels/{id}/{file}.md

### 技法缺失（craft 层）
- {缺失的通用技法} → 更新 craft/{file}.md
```
