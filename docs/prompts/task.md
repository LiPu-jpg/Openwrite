任务文档：风格文档生成循环

目标
- 通过分批读取原著，维护“风格文档”（阅读原文时生成）与“风格生成注意事项”（生成完分析后生成）。
- 同步维护“初次大纲”（阅读原文时生成）与“大纲生成注意事项”（生成完分析后生成）。
- 每读满3批，切换到“生成+差异分析”循环。

当前进度
- 批次1已完成：已输出 `style/*.md` 与 `outline/initial.md`。
- 批次2完成：已更新风格与大纲。
- 批次3进行中：正在读后续章节并更新。

执行流程（简版）
1) 读取一批文本（约30万字，分段读取）。
2) 更新：`style/*.md`、`style/summary.md`、`outline/initial.md`。
3) 记录注意事项：`outline/notes.md`。
4) 批次达到3次后进入“生成+差异分析”循环：
   - 子代理生成内容
   - 对比原著风格差异
   - 更新 `style/*.md` 与 `style/summary.md`

差异分析输出要求
- 列出“缺失的文风要素”和“过度偏离的写法”。
- 解释“为什么原先风格总结不足以约束生成”。
